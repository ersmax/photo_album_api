Web Api

-----
 Our API supports CRUD functionalities (Create, Retrieve, Update and Delete)

 single image URI endpoint:
 http://localhost:3000/api/:imageid

 Please note that in the examples below, the response example is diplayed in the "Pretty" version.




  • GET image
  • GET images
  • POST image
  • PUT image
  • DELETE image
  • Reset database

ENDPOINTS:


-----(1.1) GET single Image-----


 The GET method with id data in the request body retrieves the image datas corrisponding to the id

 Request body = none

 URL: http://localhost:3000/api/:imageId

 possible response status: 200, 400, 404, 500

 response MIME format: application/JSON

 Response example (response status 200):
 [
 {
  "id": 4,
  "author": "Tolkien",
  "alt": "Image of Tolkien",
  "tags": "English writer professor",
  "image": "https://upload.wikimedia.org/wikipedia/commons/d/d4/J._R._R._Tolkien%2C_ca._1925.jpg",
  "description": "English writer, appointed by Queen Elizabeth for academic writing skills"
 }
 ]
-----------------------------

-----(2) POST Image-----



 The POST method with "author", "alt", "tags", "description", "image" datas in the request body inserts the new image and the corresponding datas in the database with a unique id

 Request MIME format = application/JSON

 URL: http://localhost:3000/api

 Request sample
 {
  "author": "Tolkien",
  "alt": "Image of Tolkien",
  "tags": "English writer professor",
  "image": "https://upload.wikimedia.org/wikipedia/commons/d/d4/J._R._R._Tolkien%2C_ca._1925.jpg",
  "description": "English writer, appointed by Queen Elizabeth for academic writing skills"
 }

 possible response status: 201, 400, 500
 response MIME format: application/JSON (for errors)
 Response example (response status 500):
 {
  "Error": "code 500, server error occurred."
 }

-----------------------------

-----(3) PUT Image-----



 The PUT method with "author", "alt", "tags", "description", "image","id" datas in the request body updates the image corrisponding to the "id" with the new datas in the request body

 Request MIME format = application/JSON

 URL: http://localhost:3000/api/:imageId

 Request sample
 {
  "author": "Tolkien",
  "alt": "Image of Tolkien",
  "tags": "English writer professor",
  "image": "https://upload.wikimedia.org/wikipedia/commons/d/d4/J._R._R._Tolkien%2C_ca._1925.jpg",
  "description": "English writer, appointed by Queen Elizabeth for academic writing skills"
 }

 possible response status: 204, 400, 404, 500
 response MIME format: application/JSON (for errors)

 Response example (response status 500):
 {
  "Error": "code 500, server error occurred."
 }

-----------------------------

-----(4) DELETE Image-----



 The DELETE method with "id" data in the request body deletes the image corrisponding to the "id" from the database

 Request body = none

 URL: http://localhost:3000/api/:imageId

 possible response status: 204, 400, 404, 500
 response MIME format: application.JSON (for errors);

 Response example (response status 500):
 {
  "Error": "code 500, server error occurred."
 }

-----------------------------

-----(4) Reset database-----



 
resets the database deleting all images

 Request body = none

 URL: http://localhost:3000/api/reset

 possible response status: 204, 500
 response MIME format: application/JSON (for errors)

 Response example (response status 500):
 {
  "Error": "code 500, server error occurred."
 }

-----------------------------
